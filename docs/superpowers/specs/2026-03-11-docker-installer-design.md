# Docker Installer Design Spec

## Overview

A single downloadable script that deploys Maintenance Manager into Docker with zero prerequisites beyond Docker itself. Polished terminal UI with colored output, progress indicators, and step-by-step feedback. Cross-platform: bash for Linux/macOS, PowerShell for Windows.

## How Users Run It

**Linux/macOS:**
```
curl -sL https://raw.githubusercontent.com/Antonin-Bohac/MaitenanceManager/master/install.sh | bash
```

**Windows PowerShell:**
```
irm https://raw.githubusercontent.com/Antonin-Bohac/MaitenanceManager/master/install.ps1 | iex
```

Supports `--uninstall` flag for complete cleanup with confirmation prompt.

## Installer Flow

### Step 1: Banner
ASCII art banner with app name and version, using box-drawing characters:
```
╔══════════════════════════════════════════╗
║   🔧 Maintenance Manager Installer      ║
╚══════════════════════════════════════════╝
```

### Step 2: Docker Check
- Verify `docker` command exists
- Verify Docker daemon is running (`docker info`)
- If missing: print platform-specific download URL and exit
- If daemon not running: prompt user to start it and exit
- Display detected Docker version on success

### Step 3: Port Selection
- Start at port 8000, check if free
- Increment until a free port is found
- Display selected port

### Step 4: Build Image
- Create temp directory
- Write Dockerfile that clones repo from GitHub, installs deps, bundles seed DB
- Write entrypoint.sh that handles first-run seed copy + migration
- Run `docker build` with progress output
- Animated spinner/progress while building
- Clean up temp directory after build

### Step 5: Start Container
- Create named volume `maintenance-manager-data`
- Run container `maintenance-manager` with volume mount and selected port
- Wait for health check (HTTP 200 on `/`)

### Step 6: Completion Summary
Boxed summary with:
- URL to access the app
- Commands: stop, start, restart, view logs, uninstall
- Status: running/healthy

## Uninstall Mode

Triggered by `install.sh --uninstall` or `install.ps1 -Uninstall`:

1. Show what will be removed (container, image, volume)
2. Ask for confirmation (type 'yes')
3. Stop and remove container
4. Remove Docker image
5. Remove Docker volume (warns: this deletes all data)
6. Show completion message

## Seed Data

The current `maintenance.db` (96KB) is committed to the repo under `seed/maintenance.db`. The `.gitignore` entry `*.db` needs a negation `!seed/maintenance.db`.

**Entrypoint logic:**
1. If `/app/data/maintenance.db` does not exist on the volume, copy `seed/maintenance.db` to `/app/data/`
2. Run migration (`python -m app.migrate`)
3. Start uvicorn

This means: first run gets demo data, subsequent runs keep user's data, and removing the volume resets to demo data.

## Dockerfile (generated by installer)

```dockerfile
FROM python:3.12-slim

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends git && \
    git clone https://github.com/Antonin-Bohac/MaitenanceManager.git /tmp/repo && \
    cp -r /tmp/repo/app /app/app && \
    cp /tmp/repo/requirements.txt /app/ && \
    cp -r /tmp/repo/seed /app/seed && \
    apt-get remove -y git && apt-get autoremove -y && \
    rm -rf /tmp/repo /var/lib/apt/lists/*

RUN pip install --no-cache-dir -r requirements.txt

COPY entrypoint.sh /app/entrypoint.sh
RUN chmod +x /app/entrypoint.sh

ENV DATA_DIR=/app/data

EXPOSE 8000

ENTRYPOINT ["/app/entrypoint.sh"]
```

## Entrypoint Script (generated by installer)

```bash
#!/bin/bash
if [ ! -f /app/data/maintenance.db ]; then
    mkdir -p /app/data
    cp /app/seed/maintenance.db /app/data/maintenance.db
    echo "Seed database loaded."
fi
python -m app.migrate
exec uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## Terminal UI Elements

- **Box-drawing characters** for headers and summaries (`╔═╗║╚═╝`)
- **Color codes**: green for success, red for errors, yellow for warnings, cyan for info
- **Step indicators**: `[1/5]`, `[2/5]` etc. with checkmarks on completion
- **Spinner animation** during docker build (rotating `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏`)
- **Progress feedback** by parsing docker build output
- **Boxed summary** at completion with URL and commands

## Cross-Platform Notes

**Bash (Linux/macOS):**
- Port check: `ss -tuln` or `lsof -i :PORT` or `nc -z`
- Colors: ANSI escape codes
- Docker URL: Docker Desktop for macOS, docker.io packages for Linux

**PowerShell (Windows):**
- Port check: `Test-NetConnection -Port`
- Colors: `Write-Host -ForegroundColor`
- Docker URL: Docker Desktop for Windows
- Same Docker commands (Docker CLI is identical)
